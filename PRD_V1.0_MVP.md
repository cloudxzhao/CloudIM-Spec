## 2. 产品概述

### 2.1 产品背景

当前市场上即时通讯软件功能日益臃肿，用户体验逐渐复杂化。市场缺少一款专注于**轻量、纯净、桌面友好**的即时通讯工具。第一期目标是通过最小可行产品（MVP）快速验证核心社交链和通讯稳定性，为后续功能扩展奠定基础。

### 2.2 产品定位

一款专注于**Windows PC端**的轻量级即时通讯软件，提供稳定、低延迟的文字聊天体验。

### 2.3 产品目标

开发一款 Windows PC 端即时通讯软件，支持**稳定、低延迟的文字聊天**，并提供基础的好友关系链和群组功能，为后续扩展打下基础。

### 2.4 核心指标

| 指标维度   | 指标名称           | 目标值       | 衡量方式               |
| ---------- | ------------------ | ------------ | ---------------------- |
| 可靠性     | 消息发送成功率     | ≥ 99.9%      | 服务端日志统计         |
| 性能       | 消息端到端延迟     | < 200ms      | 网络监控系统           |
| 容量       | 同时在线用户数     | 5000人       | 服务端并发监控         |
| 体积       | 安装包大小         | ≤ 40MB       | 发布包文件大小         |
| 性能       | 冷启动时间         | ≤ 3秒        | 性能测试工具           |
| 资源占用   | 内存占用（聊天时）  | ≤ 150MB      | 任务管理器监控         |

### 2.5 产品范围

#### 2.5.1 包含范围
- 用户注册与登录（手机号/账号+密码）
- 个人资料设置（昵称、头像、个性签名）
- 添加好友（精确搜索ID/手机号，发送/接收好友请求）
- 单聊文字消息（发送、接收、复制、删除本地消息）
- 群聊（创建群聊、邀请好友、群成员列表、发送文字消息）
- 消息记录（本地存储，支持删除会话）
- 好友/群列表展示
- Windows原生托盘、最小化到系统托盘、消息通知

#### 2.5.2 不包含范围
- 图片、文件、语音、视频消息（仅文字）
- 音视频通话
- 朋友圈/动态
- 消息撤回、已读回执
- 消息云端漫游（仅本地记录）
- 群文件、群相册
- 多端同步（手机端、Web端暂不考虑）
- 系统管理员后台

### 4.2 用户账号模块

#### 4.2.1 注册

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| 功能描述 | 用户通过手机号和验证码完成注册，设置密码后创建账号                   |
| 用户角色 | 未注册用户                                                           |
| 前置条件 | 用户已安装客户端，网络连接正常                                       |
| 触发条件 | 用户在登录界面点击"注册"按钮                                         |
| 业务流程 | 1. 用户输入手机号<br>2. 点击获取验证码<br>3. 输入验证码和密码<br>4. 提交注册<br>5. 系统生成唯一ID，注册成功 |
| 输入字段 | 手机号（必填）、验证码（必填）、密码（必填，6-20位字母+数字）        |
| 业务规则 | 1. 同一手机号仅能注册一个账号<br>2. 验证码60秒内有效<br>3. 密码长度6-20位，必须包含字母和数字 |
| 后置条件 | 用户账号创建成功，自动跳转至登录界面                                 |
| 异常处理 | 1. 手机号已注册：提示"该手机号已注册"<br>2. 验证码错误/过期：提示"验证码错误"或"验证码已过期"<br>3. 密码格式不符：提示"密码格式不正确" |

#### 4.2.2 登录

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| 功能描述 | 用户通过账号ID/手机号和密码登录系统                                   |
| 用户角色 | 已注册用户                                                           |
| 前置条件 | 用户已注册账号，网络连接正常                                         |
| 触发条件 | 用户在登录界面输入账号和密码后点击"登录"按钮                         |
| 业务流程 | 1. 用户输入账号（ID或手机号）和密码<br>2. 可选择"记住密码"或"自动登录"<br>3. 提交登录<br>4. 验证通过后进入主界面 |
| 输入字段 | 账号（必填）、密码（必填）、记住密码（可选）、自动登录（可选）        |
| 业务规则 | 1. 支持账号ID或手机号登录<br>2. 连续5次密码错误锁定账号15分钟<br>3. 同一账号仅允许一个PC端登录，后登录踢掉前一个 |
| 后置条件 | 登录成功，进入主界面，加载好友列表和会话列表                         |
| 异常处理 | 1. 账号或密码错误：提示"账号或密码错误"<br>2. 账号被锁定：提示"账号已锁定，请15分钟后重试"<br>3. 账号在其他设备登录：提示"您的账号已在其他设备登录" |

#### 4.2.3 个人资料设置

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| 功能描述 | 用户设置和修改个人资料，包括昵称、头像、个性签名                     |
| 用户角色 | 已登录用户                                                           |
| 前置条件 | 用户已登录系统                                                       |
| 触发条件 | 用户点击头像或进入"个人设置"页面                                     |
| 业务流程 | 1. 进入个人资料页面<br>2. 修改昵称/头像/个性签名<br>3. 点击保存<br>4. 更新成功 |
| 输入字段 | 昵称（2-12字符，支持中文/英文/数字）、头像（jpg/png格式）、个性签名（0-30字符） |
| 业务规则 | 1. 昵称不能为空，长度2-12字符<br>2. 头像默认为系统默认头像<br>3. 头像显示为圆形 |
| 后置条件 | 个人资料更新成功，好友可见更新后的资料                               |
| 异常处理 | 1. 昵称过长：提示"昵称长度不能超过12个字符"<br>2. 头像格式错误：提示"请选择jpg或png格式图片" |

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| 功能描述 | 向其他用户发送好友申请，对方同意后建立好友关系                        |
| 用户角色 | 已登录用户                                                           |

| 后置条件 | 好友申请已发送，等待对方处理                                         |
| 异常处理 | 1. 已是好友：提示"该用户已是您的好友"<br>2. 申请未处理：提示"已有待处理的申请" |

## 5. 非功能需求

### 5.1 性能要求
- **响应时间**：冷启动 ≤ 3秒，热启动 ≤ 1秒
- **并发用户**：不少于5000人同时在线
- **数据处理**：滚动加载20条历史消息 ≤ 500ms
- **资源占用**：空闲时内存占用 ≤ 80MB，聊天界面打开时 ≤ 150MB
- **网络适应**：支持弱网环境（2G/3G/高延迟）下消息重试机制，超时30秒自动提示发送失败

### 5.2 安全性要求
- **身份认证**：账号密码认证，连续5次失败锁定15分钟
- **权限控制**：同一账号仅允许一个PC端登录
- **数据加密**：所有API请求使用HTTPS，长连接使用WSS（WebSocket over TLS）
- **安全防护**：敏感词过滤（政治、色情等），命中后消息发送失败
- **合规要求**：遵守《网络安全法》《个人信息保护法》相关规定

### 5.3 可用性要求
- **界面友好**：简洁白/灰背景，消息气泡区分（我方浅绿，对方浅灰）
- **操作引导**：首次使用提供简单引导
- **错误提示**：清晰明确的错误提示信息
- **帮助文档**：提供在线帮助文档链接
- **无障碍支持**：支持键盘操作

### 5.4 可靠性要求
- **系统可用性**：99.9%
- **数据备份**：服务端定期备份，客户端本地存储
- **故障恢复**：断线自动重连，重试间隔递增（5s, 10s, 30s），最多重试5次
- **容错能力**：消息发送失败可手动重发

### 5.5 可维护性要求
- **日志记录**：客户端记录关键操作日志，服务端记录API调用日志
- **监控告警**：服务端监控系统运行状态，异常告警
- **配置管理**：支持配置文件管理
- **版本升级**：支持在线更新提示

### 5.6 兼容性要求
- **浏览器兼容**：不涉及（PC客户端）
- **设备兼容**：Windows 10 64位（版本1809及以上）、Windows 11
- **分辨率适配**：最低支持1366×768，默认窗口900×600可缩放

用户(User) ── 1:N ── 好友关系(Friendship)

### 9.2 业务规则补充

1. **敏感词过滤**：消息发送前进行敏感词检测，命中敏感词的消息禁止发送并提示用户
2. **离线消息**：服务端保存离线消息最多7天，用户上线后自动拉取
3. **消息时序**：每条消息由服务端生成全局唯一自增ID，客户端按ID排序显示
